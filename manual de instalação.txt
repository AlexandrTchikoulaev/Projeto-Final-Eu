Lê este manual do início ao fim antes de começares a fazer qualquer coisa.
Segue os passos pela ordem indicada. Não saltes nenhum.

O sistema é composto por:
  - Uma API e website local (FastAPI + HTML)
  - Uma base de dados PostgreSQL (dentro de Docker)
  - Um armazenamento de ficheiros MinIO (dentro de Docker)
  - Uma interface gráfica de gestão de bases de dados (pgAdmin, dentro de Docker)

Para funcionar precisas de instalar no computador:
  1. Git
  2. Python 3.11
  3. Docker Desktop
  4. As bibliotecas Python do projeto (instaladas automaticamente)

Tempo estimado de instalação: 30 a 60 minutos (dependendo da velocidade da internet)


══════════════════════════════════════════════════════════════════════════════════
  PARTE 1 — INSTALAR O GIT E CLONAR O PROJETO
══════════════════════════════════════════════════════════════════════════════════

Git é uma ferramenta que permite descarregar o código do projeto diretamente
do GitHub (um repositório de código online). Vamos usá-la para obter todos os
ficheiros do projeto de forma automática.

PASSO 1 — Descarregar o instalador do Git.

  a) Abre o browser (Edge, Chrome, etc.)
  b) Vai ao site oficial do Git:
       https://git-scm.com/download/win
  c) O download deve começar automaticamente. Se não começar, clica em
     "Click here to download" na página.
     O ficheiro tem um nome parecido com "Git-2.xx.x-64-bit.exe"
  d) Aguarda o download terminar.


PASSO 2 — Instalar o Git.

  a) Abre o ficheiro "Git-2.xx.x-64-bit.exe" que descarregaste
     (normalmente fica na pasta Downloads)

  b) Se aparecer uma janela a pedir permissões de administrador (UAC), clica "Sim"

  c) Vai aparecendo uma série de janelas com opções. Em TODAS elas podes deixar
     as opções por defeito e clicar em "Next" (Seguinte).
     Não precisas de alterar nada.

  d) No final clica em "Install" e aguarda (1-2 minutos).

  e) Clica em "Finish" para terminar.


PASSO 3 — Verificar se o Git foi bem instalado.

  a) Carrega nas teclas  Windows + R  ao mesmo tempo

  b) Escreve "cmd" e prime Enter

  c) Na janela preta que abre, escreve:
       git --version
     e prime Enter

  d) Deve aparecer algo como:
       git version 2.xx.x.windows.x

  Se aparecer essa mensagem, o Git está instalado corretamente.


PASSO 4 — Escolher onde guardar o projeto.

  Recomenda-se guardar o projeto no Ambiente de Trabalho (Desktop) ou em C:\.
  Evita caminhos com acentos ou espaços (por exemplo, usa "C:\ProjetoFinal"
  em vez de "C:\Projeto Final").

  Para colocar no Desktop, o caminho completo vai ser algo como:
       C:\Users\TeuNome\Desktop\Projeto-Final-Eu


PASSO 5 — Clonar (descarregar) o projeto do GitHub.

  a) Abre o terminal: Windows + R → escreve "cmd" → prime Enter

  b) Navega para a pasta onde queres guardar o projeto.
     Por exemplo, para ir para o Desktop:
       cd C:\Users\TeuNome\Desktop
     (substitui "TeuNome" pelo teu nome de utilizador do Windows)

     DICA: Para saberes o teu nome de utilizador, no terminal escreve:
       echo %USERNAME%
     e prime Enter.

  c) Escreve o seguinte comando e prime Enter:
       git clone https://github.com/AlexandrTchikoulaev/Projeto-Final-Eu.git

  d) O Git vai descarregar todos os ficheiros do projeto. Vês mensagens como:
       Cloning into 'Projeto-Final-Eu'...
       Receiving objects: 100% ...
     Aguarda até terminar.

  e) No final é criada uma pasta chamada "Projeto-Final-Eu" no local escolhido.

  f) Para confirmar que correu bem, escreve:
       dir Projeto-Final-Eu
     Deves ver ficheiros como "start.py", "requirements.txt", etc.

  ATENÇÃO: Guarda bem o caminho completo desta pasta. Vais precisar dele
  mais à frente (por exemplo: C:\Users\TeuNome\Desktop\Projeto-Final-Eu).


══════════════════════════════════════════════════════════════════════════════════
  PARTE 2 — INSTALAR O PYTHON 3.11
══════════════════════════════════════════════════════════════════════════════════

Python é a linguagem de programação usada neste projeto. Precisas da versão 3.11.

PASSO 6 — Descarregar o instalador do Python 3.11.

  a) Abre o browser (Edge, Chrome, etc.)
  b) Vai ao site oficial do Python:
       https://www.python.org/downloads/release/python-3119/
  c) Nessa página, desce até à secção "Files"
  d) Clica em "Windows installer (64-bit)" — deverá ser um ficheiro com um nome
     parecido com "python-3.11.9-amd64.exe"
  e) Aguarda o download terminar

  NOTA: Se o teu computador for muito antigo (32 bits), escolhe "Windows installer
  (32-bit)". A grande maioria dos computadores modernos é 64 bits.


PASSO 7 — Instalar o Python 3.11.

  a) Abre o ficheiro "python-3.11.9-amd64.exe" que acabaste de descarregar
     (normalmente fica na pasta Downloads)

  b) VAI APARECER UMA JANELA DE INSTALAÇÃO. MUITO IMPORTANTE:
     Na parte de baixo da janela, HAZ O CHECK EM:
       ☑  "Add python.exe to PATH"
     Se não fizeres este check, o Python não vai funcionar a partir do terminal!

  c) Clica em "Install Now" (instalação recomendada, mais simples)

  d) Se aparecer uma janela a pedir permissões de administrador (UAC), clica "Sim"

  e) Aguarda a instalação terminar (pode demorar 2-3 minutos)

  f) No final aparece "Setup was successful". Clica em "Close".

  g) OPCIONAL mas recomendado: se aparecer um botão "Disable path length limit",
     clica nele. Resolve problemas com caminhos de ficheiros muito longos.


PASSO 8 — Verificar se o Python foi bem instalado.

  a) Carrega nas teclas  Windows + R  ao mesmo tempo (abre a janela "Executar")
  b) Escreve "cmd" e prime Enter
     (Abre uma janela preta chamada "Linha de Comandos" ou "Command Prompt")

  c) Na janela preta, escreve:
       python --version
     e prime Enter

  d) Deve aparecer algo como:
       Python 3.11.9

  Se aparecer essa mensagem, o Python está instalado corretamente. Se aparecer
  um erro como "não é reconhecido como um comando", significa que o passo
  "Add to PATH" não foi feito — desinstala o Python e repete o Passo 7 com atenção.


══════════════════════════════════════════════════════════════════════════════════
  PARTE 3 — INSTALAR O DOCKER DESKTOP
══════════════════════════════════════════════════════════════════════════════════

Docker é uma ferramenta que cria "contentores" — ambientes isolados onde correm
a base de dados e o armazenamento de ficheiros. É como ter pequenos servidores
a correr dentro do teu computador, sem precisar de os instalar manualmente.

PASSO 9 — Verificar os requisitos do Docker Desktop.

  O Docker Desktop requer:
    - Windows 10 versão 1903 ou superior (ou Windows 11)
    - Pelo menos 4 GB de RAM (recomendado 8 GB)
    - Virtualização ativada na BIOS (na maioria dos computadores modernos já está)

  Para verificar a versão do Windows:
    a) Carrega  Windows + R
    b) Escreve "winver" e prime Enter
    c) Verifica a versão que aparece


PASSO 10 — Descarregar o Docker Desktop.

  a) Abre o browser
  b) Vai ao site oficial do Docker:
       https://www.docker.com/products/docker-desktop/
  c) Clica no botão de download para Windows
     (deve ser um ficheiro chamado "Docker Desktop Installer.exe")
  d) Aguarda o download (o ficheiro tem cerca de 500 MB, pode demorar)


PASSO 11 — Instalar o Docker Desktop.

  a) Abre o ficheiro "Docker Desktop Installer.exe"

  b) Se aparecer uma janela UAC a pedir permissões, clica "Sim"

  c) Na janela de instalação, deixa as opções por defeito selecionadas:
       ☑  Use WSL 2 instead of Hyper-V (recomendado)
     Clica em "Ok"

  d) A instalação começa — aguarda (pode demorar 5-10 minutos)

  e) No final, clica em "Close and restart" para reiniciar o computador.
     IMPORTANTE: Guarda todo o trabalho aberto antes de reiniciar!


PASSO 12 — Configurar o Docker Desktop após reinício.

  a) Após o reinício, o Docker Desktop pode abrir automaticamente. Se não abrir,
     procura "Docker Desktop" no menu Iniciar e abre-o.

  b) Pode aparecer uma janela de boas-vindas ou um tutorial. Podes ignorar/fechar.

  c) É possível que apareça uma mensagem sobre instalar o "WSL 2 kernel update".
     Se aparecer, clica no link e instala a atualização. Depois fecha e reabre
     o Docker Desktop.

  d) Aguarda até o ícone do Docker (uma baleia azul) aparecer na barra de tarefas
     (zona do relógio, no canto inferior direito).

  e) O ícone pode ficar animado (a baleia com contentores a mover) durante o
     arranque. Espera até ficar estático — isso significa que o Docker está pronto.


PASSO 13 — Verificar se o Docker foi bem instalado.

  a) Abre uma nova janela do terminal:
       Windows + R → escreve "cmd" → prime Enter

  b) Escreve:
       docker --version
     e prime Enter

  c) Deve aparecer algo como:
       Docker version 27.x.x, build xxxxxxx

  d) Escreve também:
       docker compose version
     e prime Enter

  e) Deve aparecer algo como:
       Docker Compose version v2.x.x

  Se ambos os comandos funcionaram, o Docker está instalado corretamente.


══════════════════════════════════════════════════════════════════════════════════
  PARTE 4 — INSTALAR AS BIBLIOTECAS PYTHON DO PROJETO
══════════════════════════════════════════════════════════════════════════════════

O projeto usa várias bibliotecas Python especializadas que precisam de ser
instaladas. Estão listadas no ficheiro "requirements.txt" dentro da pasta
do projeto.

PASSO 14 — Abrir o terminal na pasta do projeto.

  Método A (mais fácil):
    a) Abre o Explorador de Ficheiros (Windows + E)
    b) Navega até à pasta do projeto (por exemplo, C:\Users\TeuNome\Desktop\Projeto-Final-Eu)
    c) Clica na barra de endereço (onde aparece o caminho, no topo)
    d) Escreve "cmd" e prime Enter
    → Abre um terminal JÁ NA pasta do projeto

  Método B (alternativa):
    a) Abre o terminal normalmente (Windows + R → cmd → Enter)
    b) Escreve o seguinte comando (substitui pelo caminho real da tua pasta):
         cd C:\Users\TeuNome\Desktop\Projeto-Final-Eu
       e prime Enter

    EXEMPLO: Se a pasta está no Desktop do utilizador "professor", o comando seria:
         cd C:\Users\professor\Desktop\Projeto-Final-Eu

  Para confirmar que estás na pasta certa, escreve:
       dir
  e prime Enter. Deves ver ficheiros como "start.py", "requirements.txt", etc.


PASSO 15 — Instalar as bibliotecas Python.

  Com o terminal aberto NA PASTA DO PROJETO, escreve o seguinte comando e
  prime Enter:
       pip install -r requirements.txt

  ATENÇÃO: Este processo pode demorar vários minutos (5 a 20 minutos dependendo
  da internet), pois vai descarregar e instalar muitas bibliotecas.

  Vais ver muitas linhas a aparecer com "Downloading..." e "Installing...".
  É normal. Não feches o terminal.

  No final deve aparecer algo como:
       Successfully installed fastapi-x.x uvicorn-x.x ...

  Se aparecer algum erro a vermelho, lê a mensagem. Os erros mais comuns são:
    - "pip is not recognized": o Python não foi instalado corretamente (repete PASSO 7)
    - "No module named pip": tenta com "python -m pip install -r requirements.txt"
    - Erros de rede: verifica a ligação à internet e tenta novamente


══════════════════════════════════════════════════════════════════════════════════
  PARTE 5 — CONFIGURAÇÃO INICIAL DO SISTEMA (SÓ FAZER UMA VEZ)
══════════════════════════════════════════════════════════════════════════════════

Este passo cria as bases de dados, configura o armazenamento de ficheiros e
prepara tudo para a primeira utilização. SÓ PRECISAS DE FAZER ISTO UMA VEZ.

PASSO 16 — Garantir que o Docker Desktop está a correr.

  a) Olha para a barra de tarefas (zona do relógio, canto inferior direito).
  b) Deve aparecer o ícone do Docker (baleia azul).
  c) Se não aparecer, procura "Docker Desktop" no menu Iniciar e abre-o.
  d) Aguarda até o ícone ficar estático (não animado) — o Docker está pronto.


PASSO 17 — Executar o script de configuração inicial.

  a) Abre o terminal NA PASTA DO PROJETO (como explicado no Passo 14)

  b) Escreve o seguinte comando e prime Enter:
       python Codes\Setup\setup.py

  c) O script vai fazer automaticamente (pode demorar 5-15 minutos):
       - Verificar se o Docker está disponível
       - Iniciar os contentores (base de dados, pgAdmin, MinIO)
       - Aguardar que os serviços fiquem prontos
       - Criar as bases de dados necessárias
       - Criar as tabelas
       - Executar o ETL (processar os dados)

  d) Vais ver muitas mensagens no terminal. É normal. Mensagens com "OK" ou
     "concluído" são boas. Mensagens com "ERRO" indicam problemas.

  e) No final, se correr tudo bem, aparece:
       ╔══════════════════════════════════════════════════════════╗
       ║         SETUP CONCLUÍDO COM SUCESSO                       ║
       ╚══════════════════════════════════════════════════════════╝

  ATENÇÃO: Se aparecer um erro durante este passo, não entres em pânico.
  Lê a mensagem de erro com atenção e verifica:
    - O Docker Desktop está a correr? (ícone na barra de tarefas)
    - Estás na pasta certa? (usa "dir" para confirmar)
    - As bibliotecas Python foram instaladas? (repete o Passo 15)

PASSO 18 — Iniciar o sistema.

    - pgAdmin:       http://localhost:5051

══════════════════════════════════════════════════════════════════════════════════
  PARTE 7 — CONFIGURAR CHAVE API (SE NECESSÁRIO PARA O CHAT INTELIGENTE)
══════════════════════════════════════════════════════════════════════════════════

O sistema tem uma funcionalidade de chat inteligente que usa inteligência
artificial (LangChain). Para essa funcionalidade funcionar, pode ser necessário
configurar uma chave de API.

PASSO 21 — Configurar a chave de API (se pedido).

  Se ao usar o chat aparecer um erro sobre "API key" ou "authentication":

  a) O aluno deve fornecer a chave de API necessária (OpenAI ou outra).

  b) Para configurar, cria uma variável de ambiente no Windows:
       - Carrega Windows + R, escreve "sysdm.cpl" e prime Enter
       - Clica no separador "Avançadas"
       - Clica em "Variáveis de Ambiente"
       - Em "Variáveis do sistema", clica em "Novo"
       - Nome: OPENAI_API_KEY  (ou o nome que o aluno indicar)
       - Valor: (a chave fornecida pelo aluno)
       - Clica OK em todas as janelas

  c) Fecha e reabre o terminal para as alterações terem efeito.


══════════════════════════════════════════════════════════════════════════════════
  RESOLUÇÃO DE PROBLEMAS COMUNS
══════════════════════════════════════════════════════════════════════════════════

PROBLEMA: "python não é reconhecido como um comando interno ou externo"
SOLUÇÃO:  O Python não foi adicionado ao PATH. Desinstala o Python (via
          "Adicionar ou Remover Programas") e repete o PASSO 3, desta vez
          garantindo que marcas a opção "Add python.exe to PATH".

----------

PROBLEMA: "docker não é reconhecido como um comando interno ou externo"
SOLUÇÃO:  O Docker Desktop não está a correr ou não foi instalado corretamente.
          Abre o Docker Desktop pelo menu Iniciar. Se não aparecer, reinstala-o
          (PASSO 6 e 7).

----------

PROBLEMA: O Docker Desktop não inicia / fica sempre "a carregar"
SOLUÇÃO A: Reinicia o computador e tenta novamente.
SOLUÇÃO B: Pode ser necessário ativar a virtualização na BIOS. Contacta o aluno
           ou o suporte de informática da instituição.
SOLUÇÃO C: Desinstala e reinstala o Docker Desktop.

----------

PROBLEMA: Erro "port is already in use" ou "porta já em uso"
SOLUÇÃO:  Algum serviço no computador está a usar a mesma porta. Reinicia o
          computador e tenta novamente. Se persistir, contacta o aluno.

----------

PROBLEMA: O website abre mas não carrega dados / aparece em branco
SOLUÇÃO:  O setup inicial pode não ter corrido corretamente. Tenta correr
          novamente:  python Codes\Setup\setup.py

----------

PROBLEMA: "No module named 'fastapi'" ou outro módulo em falta
SOLUÇÃO:  As bibliotecas não foram instaladas. Repete o PASSO 11.

----------

PROBLEMA: O sistema fica lento ou o computador fica pesado
EXPLICAÇÃO: O Docker está a correr vários serviços em simultâneo, o que consome
            memória RAM. É normal alguma lentidão. Quando não estiveres a usar
            o sistema, para-o sempre (PASSO 16).

----------

PROBLEMA: Perdeste ou não consegues encontrar a pasta do projeto
SOLUÇÃO:  Usa o Windows Search (tecla Windows, escreve "start.py") para
          encontrar o ficheiro e descobrir onde está a pasta.
